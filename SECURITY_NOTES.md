# Sprint 7: Security and Reliability Notes

Дата обновления: `2026-03-04`

## Реализованные меры (чеклист)
- [x] API токен для signaling (`SIGNAL_API_TOKEN` + заголовок `x-nexus-token`), кроме `/health`.
- [x] Защищен `/metrics` тем же токеном.
- [x] Глобальный rate limit на окно времени.
- [x] Анти-спам защита `register` по одному ключу (cooldown 2 сек).
- [x] Ретраи с exponential backoff в клиенте для signaling HTTP операций.
- [x] Обработка сетевых обрывов в `watch-session` с автоматическим переподключением.
- [x] Нагрузочный concurrent-тест для `/v1/register` (допускаются `200` и `429`, без `5xx`).

## Privacy review (логи и CLI-вывод)
- [x] Личный ключ в `show-key` маскируется (`NX-***..`).
- [x] Ключ в `resolve` выводится в маскированном виде.
- [x] Ответы API в логах клиента проходят санитизацию:
  - поля `*key` маскируются;
  - `sdp*`, `candidate*`, `endpoint*` редактируются как `***`;
  - длинные тела ответов обрезаются.
- [x] Добавлен unit-тест на редактирование чувствительных полей в логе.

## Остаточные риски
- Токен передается как static header; для production нужен TLS termination (HTTPS) и ротация секрета.
- Текущий rate limit глобальный; для публичного деплоя лучше заменить на per-IP/per-key + burst policy.
